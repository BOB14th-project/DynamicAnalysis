## 수집 가능한 정보 범위

### 현재 수집하는 정보
1. 함수 레벨 추적
- 함수명, 모듈명, 호출 순서
- 모든 함수 인자값 (포인터 주소, 스칼라 값)
- 반환값
- 호출 스택 (backtrace)

2. 메모리 데이터 완전 수집
- 포인터가 가리키는 메모리 내용 (hexdump)
- ASCII 문자열 변환
- 데이터 길이 정보
- 메모리 영역 분류 (코드/데이터/스칼라)

3. 암호화 특화 정보
- OpenSSL EVP API 상세 분석
- 암호화 컨텍스트 내부 구조
- 키/IV/평문/암호문 실제 데이터
- 암호화 알고리즘 이름, 블록 크기
- 암호화/복호화 방향

4. 동적 라이브러리 완전 추적
- 런타임 라이브러리 로딩 감지
- 지연 로딩된 함수들 자동 후킹

- 코드 복잡도, 의존성 분석

2. 실행되지 않은 코드 경로 (**별도 도구 필요**) - ??????
- 조건부 실행 블록
- 예외 처리 코드
- 다른 입력값에 따른 경로

- **전략:** 정적분석으로 전체 코드 식별 → 동적분석으로 실행된 부분 추적(현재 도구 사용) → 차집합으로 미실행 부분 발견 -> 퍼징으로 새로운 입력 생성해서 미실행 블록 도달 시도

3. 하드웨어/커널 레벨 (**별도 도구 필요**) - ???????
- CPU 캐시, 타이밍 공격 정보
- 하드웨어 암호화 가속기 사용
- 커널 레벨 시스템 콜 상세
- **추적 도구:** 
  - **성능/캐시:** Intel VTune, perf, Intel PT(Processor Trace)
  - **시스템 콜:** strace(Linux), Process Monitor(Windows), eBPF
  - **하드웨어 암호화:** Intel SDE, AES-NI 탐지, HSM/TPM 추적
  - **타이밍 공격:** Chronos, 사이드채널 분석 프레임워크

```cpp
# CPU 성능 카운터로 암호화 함수 분석
perf record -e cache-misses,branches,instructions ./target
perf report

# 시스템 콜 추적
strace -f -e trace=read,write,openat,mmap ./target

# Intel PT로 완전한 CPU 실행 추적
perf record -e intel_pt/cyc=1/u ./target
perf script --itrace=i1usecge

# AES-NI 하드웨어 가속 확인
grep aes /proc/cpuinfo

```

### 의문점 및 답변

#### Q1. 결과물의 양이 많아서 판단이 어려운 문제
**A:** 현재는 필터링으로 해결했지만, 더 나은 방법들:
- **실시간 휴리스틱 필터링**: 엔트로피 기반 데이터 분류
- **패턴 기반 분류**: 메모리 접근 패턴, 함수 호출 빈도 분석
- **머신러닝 기반 탐지**: 암호화 함수 행동 시그니처 학습

#### Q2. 난독화된 상황에서 암호화 함수 판별 방법
**A:** 동적 분석의 강력한 장점 - **함수명 몰라도 행동으로 식별 가능**:
- **데이터 엔트로피 분석**: 입력 대비 출력의 높은 엔트로피 = 암호화
- **메모리 접근 패턴**: 16/24/32바이트 키 크기, S-box 테이블 접근
- **반복 연산 패턴**: 블록 암호화의 라운드 함수 반복
- **비트 연산 집약도**: XOR, 회전, 치환 연산의 집중도

#### Q3. 난독화 시 함수명 가시성과 대응책
**함수명이 안 보이는 경우:**
- 스트립된 바이너리 (stripped binary)
- 정적 링크된 라이브러리  
- 커스텀 암호화 구현

**대응책:**
- **함수 주소 기반 추적**: 심볼 없어도 주소로 식별
- **메모리 패턴 매칭**: 알려진 암호화 상수 탐지
- **행동 시그니처**: 입출력 변환 패턴으로 알고리즘 추정

#### Q4. 동적 분석으로 가져와야 할 핵심 정보
- **메모리 I/O 패턴**: 입력 → 변환 → 출력 흐름
- **데이터 엔트로피 변화**: 평문 → 암호문 엔트로피 증가
- **특정 크기 메모리 블록**: AES(16), 3DES(8), RSA(256+) 바이트
- **반복 연산 패턴**: 암호화 라운드 함수의 반복
- **상수 테이블 접근**: S-box, 치환 테이블 참조

#### Q5. 동적 분석이 불가능한 상황
**기술적 한계:**
- 하드웨어 기반 암호화 (HSM, TPM, AES-NI 내부)
- 커널 모드 전용 암호화
- 강력한 안티 디버깅/가상화 탐지
- 시간 지연 기반 우회 탐지

**환경적 한계:**
- 네트워크 격리 환경 (에어갭)
- 특정 하드웨어 의존성 (스마트카드 등)
- 라이선스/인증 서버 의존
- 실시간/임베디드 시스템 (성능 영향)
